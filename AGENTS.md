# MemScope 开发指南

> 本文件定义了 MemScope 项目的开发规范和约束，所有开发工作必须遵守。

## 赛题要求

**必读文件**：[.hermes/competition_requirements.md](.hermes/competition_requirements.md)

在开发 MemScope 的任何功能前，必须先阅读赛题要求文档，确保：
1. 所有功能开发对齐赛题的三大挑战（定义记忆、构建引擎、证明价值）
2. 评测数据集覆盖赛题要求的三种测试（抗干扰、矛盾更新、效能验证）
3. 最终交付物包含白皮书、可运行 Demo、评测报告

## 项目目标（核心要求）

> ⚠️ 以下6点是项目的明确目标，所有研发工作必须围绕这些目标展开。

### 目标1：评测数据集全面评估真实性能
- 评测数据集能根据赛题要求全面评估 MemScope 真实性能
- 必须覆盖四大方向：CLI命令记忆、飞书决策记忆、个人偏好记忆、团队知识健康
- 必须包含三项必测：抗干扰测试、矛盾更新测试、效能指标验证

### 目标2：评测数据集符合主流规范
- 每个样本的格式和内容能够符合主流 memory 评测数据集的设置规范
- 参考 LongMemEval (ICLR 2025)、LOCOMO (ACL 2024) 等学术基准
- 样本结构应包含：test_id, name, difficulty, setup, query, expected

### 目标3：长时序多轮对话样本
- 评测数据集能够包含长时序多轮对话样本以检验 memory 架构的能力
- 包含短时序（1天-1周）、中时序（1周-3个月）、长时序（3个月-2年）
- 长时序样本应有足够的干扰项（"大海捞针"场景）

### 目标4：真实评测执行与报告输出
- 评测代码能够实现将 MemScope 与 Hermes Agent 或 OpenClaw 连接
- 基于评测数据集的所有样本在飞书 CLI 上做实际评测
- **实际评测 = 根据样本输入在飞书环境里做具体执行，形成执行轨迹，对执行轨迹做评估**
- 评测代码正确设置评测指标计算（覆盖率、准确率、召回率、F1、效率指标等）
- 每次运行评测都能自动产出一份评测性能报告供后续优化

> ⚠️ **严重警告**：现有 README 里的评测结果只展示"通过了多少个评测样本"而非具体指标（覆盖率、准确率等），这是严重的错误！
> 
> **性能评估 ≠ 代码测试**：
> - 代码测试（test/）：检查函数是否抛异常、返回值是否非空 → 用 pytest
> - 性能评估（eval/）：检查返回值的语义正确性、计算具体指标 → 用评测框架
> 
> 如果评估代码只统计 pass/fail 数量而不计算 Hit Rate、Precision、Recall、F1 等具体指标，那就是在用代码测试的思路做性能评估，这是严重错误！

### 目标5：MemScope 真正连接 Hermes Agent
- MemScope 真能连接到 Hermes Agent 并替代其原有记忆架构
- 实现记忆提取、存储、检索、更新、遗忘等完整生命周期
- 通过 `ctx.register_memory_provider()` 注册，实现 `prefetch()` / `sync_turn()` / `on_session_end()` 生命周期

### 目标6：CLI 与飞书端自由切换
- MemScope 基于 Hermes Agent 和飞书 API 可通过 CLI 终端和飞书端进行交互演示
- CLI 终端和飞书端可以自由切换
- 支持飞书互动卡片推送（决策卡片、知识健康预警）

## 核心原则

### 代码测试 vs 评估（严格区分）

- **代码测试**（test/ 目录）：用 pytest + mock 检验代码是否有 bug
  - 代码测试结果 **禁止** 出现在 README 或任何对外文档中
  - 代码测试仅用于开发阶段的质量保障

- **评估**（eval/ 目录）：用真实系统 + 评测数据集衡量实际性能
  - 评估结果是对外展示性能的 **唯一数据来源**
  - 所有评估数据必须来自 `eval/real_evaluation.py` 的实际运行结果
  - **禁止编造任何评估数字**
  - **必须计算具体指标**：Hit Rate、Precision、Recall、F1、操作节省率、字符节省率、时间节省率、延迟等
  - **禁止只展示"通过样本数"**：必须展示覆盖率、准确率等具体性能指标

### 数据真实性

- 所有对外展示的数字必须有实际运行记录可追溯
- 未执行的评估标注为"待测试"，不得编造数字
- 评估结果必须包含 200 条完整评测数据集的全部结果，不得选取部分数据集

## 评测指标要求

每次评测必须计算并报告以下指标：

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 命中率 Hit Rate | >= 85% | 搜索结果中包含目标信息的比例 |
| 精确率 Precision | >= 85% | 搜索结果中相关信息的比例 |
| 召回率 Recall | >= 90% | 目标信息被检索到的比例 |
| F1-Score | >= 87% | 精确率和召回率的调和平均 |
| 操作节省率 | >= 50% | 记忆推荐减少的操作步骤比例 |
| 字符节省率 | >= 60% | 记忆推荐减少的输入字符比例 |
| 时间节省率 | >= 40% | 记忆推荐节省的时间比例 |
| 写入延迟 P50 | <= 200ms | 记忆写入操作的中位延迟 |
| 查询延迟 P50 | <= 300ms | 记忆查询操作的中位延迟 |

## 开发检查清单

在提交任何代码变更前，确认：

- [ ] 运行 `python3 -m pytest test/ -v` 确保代码测试通过
- [ ] 运行 `python3 eval/real_evaluation.py` 确保评估结果不退化
- [ ] README 中的数据全部来自真实评估结果
- [ ] README 中展示的是具体指标（Hit Rate、Precision、Recall、F1 等），而非"通过样本数"
- [ ] 没有在 README 中展示代码测试通过率
- [ ] 评测代码使用真实系统而非 mock
- [ ] 评测代码能连接 Hermes Agent 或 OpenClaw 执行实际评测
- [ ] 每次评测自动生成性能报告
