## 🔄 完整用户旅程

### 第一阶段：个人偏好建立（方向 C）

#### 场景 1.1：张三的早晨工作流

**时间**: 周一上午 9:00

张三打开飞书，对 Hermes Agent 说：

> "帮我记录一下，我通常在早上 9 点到 12 点之间做深度编码工作，这个时间段不要安排会议。"

**系统处理流程**:

```
1. 📨 接收消息 → FeishuMessageReceiver
2. 🧠 记忆提取 → MemoryExtractor
   ├─ 分类: preference
   ├─ 关键词: "喜欢"、"早上"、"不要安排会议"
   └─ 置信度: 0.75
3. 💾 存储偏好
   ├─ preference_set("zhangsan", "schedule", "deep_work_time", "09:00-12:00")
   ├─ 来源: user_explicit
   └─ 置信度: 0.95 (显式声明)
4. ✅ 确认存储
```

**输出结果**:
```
✓ 偏好已记录: schedule/deep_work_time = 09:00-12:00
  置信度: 0.95 (显式声明)
  未来安排会议时会自动避开此时间段
```

#### 场景 1.2：张三的工具偏好

张三继续说：

> "我推荐使用 VSCode 配合 Remote-SSH 插件进行远程开发，比 Vim 好用多了。"

**系统处理**:
```
1. 提取偏好: tool_preference/vscode_remote = "推荐使用"
2. 同时记录偏好: tool_preference/ide = "VSCode"
3. 来源: user_explicit, 置信度: 0.95
```

#### 场景 1.3：习惯推断（自动）

系统在后台分析张三的 500 条工具日志，自动推断行为模式：

**时间模式分析**:
```
输入: 500 条工具日志时间戳
分析: 按小时统计活跃度
输出: time_pattern — 最活跃时段 09:00-12:00, 14:00-17:00
      置信度: 0.88 (样本充足)
```

**工具频率分析**:
```
输入: 500 条工具调用记录
统计: terminal=180, browser=95, editor=85, git=72, docker=45
输出: tool_frequency — 最常用工具: terminal (180次)
      置信度: 0.82
```

**推断偏好**（自动写入）:
```
preference_set("zhangsan", "schedule", "peak_active_hours", "09:00-12:00, 14:00-17:00", source="habit_inference")
preference_set("zhangsan", "tool_preference", "most_used_tool", "terminal", source="habit_inference")
```

**冲突处理**: 如果推断的偏好与显式偏好冲突，显式优先：
```
规则: user_explicit (0.95) > habit_inference (0.82)
结果: 保留用户显式声明的值
```

## 🎯 演示亮点总结

1. **完整记忆生命周期**: 从消息接收 → 提取 → 存储 → 衰减 → 预警的全链路
2. **智能偏好推断**: 基于 500 条行为数据自动发现工作习惯
3. **团队级知识治理**: 10 大领域自动覆盖分析 + 单点故障识别
4. **冲突处理机制**: 显式声明 > 行为推断，确保用户意图优先
5. **飞书原生集成**: 互动卡片推送，无缝融入工作流
6. **零外部依赖**: 纯 Python 标准库，SQLite 存储，即插即用
