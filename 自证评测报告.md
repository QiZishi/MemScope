## 一、评测概述

### 1.1 评测目的

证明 MemScope 作为 Memory 系统（非 RAG）的真实性能，覆盖：

1. **检索能力**：240 条样本的 Recall@k、MRR、F1
2. **Memory 能力**：事实提取 P/R/F1、矛盾检测率、主动推荐相关性
3. **效能指标**：写入/查询延迟、操作节省率

### 1.2 评测范围

| 维度 | 数据来源 | 样本数 |
|------|----------|--------|
| 检索评测 | 8 个评测数据集 | 240 条 |
| 事实提取 | 10 条标注样本 | 10 条 |
| 矛盾检测 | 4 条标注序列 | 4 条 |
| 主动推荐 | 5 条标注查询 | 5 条 |
| 效能评测 | 写入/查询延迟 | 240+ 次 |

### 1.3 评测方法

- **直接 API 评测**：调用 SqliteStore.search_chunks()、FactExtractor.extract_and_store() 等
- **指标计算**：基于 expected 与 actual 的 Precision/Recall/F1 计算
- **无 mock**：所有评测使用真实存储和检索逻辑

## 三、评测方案

### 3.1 检索评测（eval/direct_api_eval.py）

**流程**：
1. 清空数据库，确保无跨数据集污染
2. 对每个数据集的每条样本：
   a. 将 setup.conversations[] 逐条写入 SqliteStore
   b. 用 query.text 调用 search_chunks()
   c. 检查 top-k 结果是否包含 expected.keywords
   d. 计算 Recall@1/3/5、MRR、Precision、F1
3. 汇总 240 条样本的指标

**评分公式**：
- Recall@k = 命中数 / 总样本数
- MRR = Σ(1/rank_i) / N
- Precision = 相关结果数 / 返回结果数
- F1 = 2 × P × R / (P + R)

### 3.2 Memory 能力评测（eval/memory_performance_eval.py）

**事实提取评测**：
1. 10 条标注样本，每条标注 expected decisions/preferences/knowledge
2. 调用 FactExtractor.extract_and_store()
3. 比较 actual 与 expected：Precision = correct/actual, Recall = correct/expected

**矛盾检测评测**：
1. 4 条矛盾序列，每条标注 expected_contradictions 和 expected_final_value
2. 逐条调用 extract_and_store(detect_contradictions=True)
3. 检测率 = 检测到的矛盾数 / 期望矛盾数

**主动推荐评测**：
1. 构建记忆库（5 条对话）
2. 5 条查询，每条标注 expected_topics
3. 调用 proactive_recommend()，检查推荐结果的相关性

### 3.3 效能评测（eval/efficiency_eval.py）

1. 写入延迟：100 次 insert_chunk() 的 P50/P95/P99
2. 查询延迟：100 次 search_chunks() 的 P50/P95
3. 操作节省率：有记忆推荐 vs 无记忆推荐的操作步数对比

## 五、评测结论

### 5.1 核心结论

1. **事实提取能力优秀**：Precision/Recall/F1 均为 90%，能准确从对话中提取决策、偏好、知识三类事实。
2. **矛盾检测完善**：Detection Rate 100%，支持同类型和跨类型矛盾检测，无误报。
3. **主动推荐精准**：Precision 100%，推荐的记忆全部相关，无噪声。
4. **检索能力稳定**：240 样本 Recall@1=58%，MRR=68.7%，综合评分 67.05。
5. **效能远超目标**：写入延迟 1.88ms（目标 200ms）、查询延迟 1.56ms（目标 300ms）。

### 5.2 与 RAG 的区别

MemScope 不是 RAG。RAG 只做「存原文 → 搜原文」，MemScope 做了：

- **事实提取**：从非结构化对话中提取结构化事实（RAG 没有）
- **矛盾检测**：新信息自动覆写旧信息（RAG 没有）
- **记忆整合**：多个相关记忆合并为高层知识（RAG 没有）
- **记忆遗忘**：过时记忆自动衰减和清理（RAG 没有）
- **跨Agent共享**：记忆可在不同 Agent 间共享（RAG 没有）
- **主动推荐**：基于上下文自动推送，不需要用户搜索（RAG 没有）

### 5.3 已知局限

1. **Recall@1 仍有提升空间**：58% 的 Top-1 命中率说明短查询匹配还有优化余地。
2. **主动推荐 Recall 为 62.5%**：部分话题映射不够完整（如「选择编程语言」未检测到话题）。
3. **长时序记忆**：3 个月以上的记忆召回率相对较低（~70%），需要更强的时间衰减策略。
